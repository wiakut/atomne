# Alias Party Game — Технічне Завдання (v1.0)

## Загальний опис

Повноцінна веб-гра в стилі Alias для компанії друзів. Гра на **одному пристрої** (телефон передається між командами). 4 тематичних раунди зі свайп-механікою, таймером, підрахунком очок і лідербордом.

---

## Стек технологій

- **React** (SPA)
- **Tailwind CSS** (стилізація)
- **Framer Motion** (анімації)
- **localStorage** (збереження прогресу, рахунку, кастомних пачок слів)
- **PWA** (manifest + service worker для встановлення на телефон)

---

## Дизайн: Cyber-Neon

- Темний фон: `#0f172a`
- Неонові рамки/акценти: cyan (`#06b6d4`) та pink (`#ec4899`)
- Glow-ефекти на кнопках і картках
- Типографіка: **Inter** або System Sans — чиста, читабельна, класична
- Загальна естетика: темний, стильний, але не перевантажений. Акцент на читабельності слів на картках.

## Екрани та Flow

### 1. Головний екран (Setup)

- **Кількість команд** (мін. 2, макс. — рекомендовано до 6)
- **Назви команд**: кнопка "Згенерувати" для випадкових смішних/крутих назв + можливість ввести вручну
- **Таймер раунду**: поле вводу кількості секунд (дефолт — 60с)
- **Розширені налаштування** (згортається секція):
  - Вмикати/вимикати окремі раунди (всі 4 увімкнені за замовчуванням)
  - Кількість слів, які кожна команда має ввести в Раунді 1 (дефолт — 10)
- Кнопка **"Почати гру"**

### 2. Екран правил раунду

Перед кожним раундом — екран з:
- Назвою раунду та його номером
- Коротким описом правил (що робити: пояснювати словами / жестами / англійською)
- Кнопка **"Старт"** для переходу до гри або введення слів

### 3. Введення слів (тільки Раунд 1)

Flow введення слів для Раунду 1 «Дай рєспєкт»:

1. Екран **"Черга команди [Назва]"** — показує яка команда зараз вводить слова. Кнопка **"Ввести слова"**
2. Екран **введення слів** — текстове поле / список, куди команда вводить задану кількість слів (з конфігурації). Кнопка **"Передати далі"**
3. Повертається екран "Черга команди" для наступної команди
4. Коли всі команди ввели — перехід до ігрової фази

**Важливо:** слова від усіх команд перемішуються в одну спільну пачку. Під час гри будь-якій команді може випасти будь-яке слово.

### 4. Ігровий екран (Play)

- **Картка зі словом** — великий, чіткий текст по центру екрану
- **Свайп-механіка** (Tinder-style):
  - **Свайп вгору** = слово вгадано → +1 бал поточній команді → наступне слово
  - **Свайп вниз** = skip → 0 балів, без штрафів → наступне слово
- **Таймер** — відображається збоку або зверху, стильна рамка
  - Коли залишилось **< 10 секунд**: цифри стають **яскраво-червоними** і **пульсують**
  - Останні 10 секунд: **звук тікання** (кожну секунду)
  - Коли таймер = 0: **фінальний біп/alarm**
- **Після закінчення таймера**:
  - Поточне слово **залишається на екрані**
  - Гравець може зробити останній свайп (вгору або вниз)
  - Свайп вниз — skip, слово не зараховується, хід команди завершується
  - Свайп вгору — слово вгадано, +1 бал, хід команди завершується
  - **Виняток — Раунд 3:** після закінчення таймера, якщо свайп вгору — з'являється **UI вибору команди** (хто саме відгадав "глобальне" слово). Детальніше в описі Раунду 3.

### 5. Лідерборд між раундами

Після того як всі команди відіграли свій хід у раунді:
- Таблиця результатів із поточним рахунком
- Glowing progress bars для візуалізації рахунку
- Кнопка **"Наступний раунд"** / **"Наступна команда"**

### 6. Фінальний екран

Після завершення всіх раундів:
- Фінальний лідерборд з підсумковим рахунком
- **Анімація конфеті / перемоги** для команди-переможця
- Кнопка **"Нова гра"**

## Раунди

### Раунд 1: «Дай рєспєкт»

- **Фаза введення:** кожна команда по черзі вводить слова (телефон передається по колу)
- **Завдання:** пояснити слово будь-яким способом (словами, жестами — без обмежень)
- **Слова:** введені гравцями. Всі слова від усіх команд перемішуються в одну спільну пачку.
- **Механіка:** стандартний свайп вгору/вниз

### Раунд 2: «Запускаю атомне»

- **Завдання:** пояснити слово **ТІЛЬКИ жестами/пантомімою** (без слів)
- **Слова:** вбудований словник українських іменників (смішних/челенджових). **1000+ слів.**
- **Механіка:** стандартний свайп вгору/вниз

### Раунд 3: «Проектор-Ж»

- **Завдання:** класичний Alias — пояснити словами (без однокореневих, без прямих перекладів)
- **Слова:** вбудований словник українських слів. **1000+ слів.**
- **Спеціальне правило — "Глобальне" слово:**
  - Коли таймер закінчується, поточне слово стає "глобальним"
  - Його може відгадати **будь-яка команда** (не тільки та, що зараз грає)
  - Після свайпу вгору на глобальному слові — з'являється **UI вибору команди**: список усіх команд + кнопка для кожної, щоб нарахувати бал тій, хто відгадала
  - Якщо ніхто не відгадав — кнопка **"Ніхто"** або свайп вниз (skip)
  - **Це правило діє ТІЛЬКИ в Раунді 3**

### Раунд 4: «Ухилянт»

- **Завдання:** пояснити англійське слово **використовуючи тільки англійську мову**
- **Слова:** вбудований словник простих англійських іменників (**elementary** рівень). **1000+ слів.**
- **Механіка:** стандартний свайп вгору/вниз

---

## Порядок ходів

Всередині кожного раунду команди грають **по черзі**:
1. Екран "Хід команди [Назва]" — підготовка, передача телефону
2. Кнопка "Готові" → запуск таймера + перша картка
3. Команда грає свій хід (таймер + свайпи)
4. Після завершення ходу — короткий підсумок (скільки слів вгадано)
5. Перехід до наступної команди
6. Коли всі команди відіграли — лідерборд раунду → наступний раунд

---

## Звук

- **Тікання** останніх 10 секунд таймера (звук кожну секунду)
- **Фінальний біп/alarm** коли таймер = 0
- Звуки генеруються через **Web Audio API** (без зовнішніх аудіо-файлів)

---

## Збереження (localStorage)

- Поточний стан гри (раунд, команди, рахунок, черга ходу)
- Введені слова (Раунд 1)
- При оновленні сторінки / закритті браузера — гра відновлюється з того ж місця

---

## PWA

- `manifest.json` з назвою гри, іконками, кольорами теми (`#0f172a`)
- Service Worker для офлайн-роботи та можливості "Add to Home Screen" на телефоні
- Після першого завантаження гра працює повністю офлайн

---

## Словники (захардкоджені)

Всі словники вбудовані в код як масиви. Не використовувати зовнішні API.

### Раунд 2 — Українські іменники для пантоміми (1000+ слів)
Категорії: тварини, професії, побутові предмети, їжа, транспорт, спорт, природа, одяг, емоції (як дія), казкові/мемні персонажі, музичні інструменти, будівлі, техніка тощо.
Слова мають бути **різної складності** — від простих (кіт, лікар, стілець) до челенджових (бюрократія, гравітація, дипломатія).
Слова мають бути цікавими для показу жестами.

### Раунд 3 — Українські слова для класичного Alias (1000+ слів)
Ширший набір: абстрактні поняття, дії, предмети, явища, професії, їжа, культура, наука.
Мають бути цікавими та різноманітними для пояснення словами.

### Раунд 4 — Англійські іменники, elementary рівень (1000+ слів)
Прості, загальновідомі англійські слова: apple, house, dog, car, school, rain, window, chair, phone, book тощо.
Рівень: **elementary** — щоб навіть ті, хто знає англійську на базовому рівні, могли пояснити.

**Повторне використання слів між іграми допускається** — не треба трекати вже використані. Слова рандомізуються (shuffle) на початку кожної гри.

## Генератор назв команд

Список смішних/крутих випадкових назв українською. Мінімум **30 варіантів**.
При натисканні "Згенерувати" — кожній команді присвоюється випадкова назва зі списку (без повторів).

Приклади стилю: "Кіберкозаки", "Шаурма Team", "Ctrl+Козак", "Борщ & Біти", "404 Команду не знайдено", "Капібари", "Дрони добра" тощо.

## Технічні вимоги

- **Mobile-first** дизайн (основний пристрій — телефон)
- Оптимізований **touch/swipe** для мобільних
- Плавні анімації (Framer Motion): свайпи карток, переходи між екранами, конфеті
- Відсутність затримок при свайпах і переходах
- Працює **офлайн** після першого завантаження (PWA + захардкоджені словники)
- **Без зовнішніх API** — вся логіка та дані всередині додатку
- **Без PIN-коду** — відкритий доступ
- **Деплой на Vercel** — проект має бути налаштований для деплою на Vercel (Next.js або Vite + відповідний `vercel.json` якщо потрібно)
